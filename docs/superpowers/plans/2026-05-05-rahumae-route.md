# Rahumäe Route + Scrollable Mobile Tab Bar — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a third route pair (Rahumäe ↔ Tallinn) and convert the route tab bar into a horizontally-scrollable, content-sized strip with an edge-fade hint, scrolling the active tab into view on render.

**Architecture:** All changes are inside the single `RailScraper.py` file. The default config in `load_config()` gains the new pair and corrected coordinates. The HTML generated by `generate_html()` wraps the tab bar in a sticky `.tab-bar-wrap` element and gives the inner `.tab-bar` `overflow-x: auto` with content-sized tabs (`flex: 0 0 auto`). The wrapper carries a `::after` gradient that toggles via a `.scrolled-end` class on scroll. `renderTabs()` calls `scrollIntoView({ inline: 'center' })` on the active tab so the restored route is always visible.

**Tech Stack:** Python 3 (Selenium + BeautifulSoup), vanilla JS, CSS. No build step. No tests in the repo — verification is manual browser checks at desktop and 375px mobile widths.

**Spec:** `docs/superpowers/specs/2026-05-05-rahumae-route-design.md`

## Notes for the implementer

- **Single file:** all code edits happen in `RailScraper.py`. No new files.
- **HTML/CSS/JS lives inside a Python f-string** (`generate_html()`, RailScraper.py:219-749). Triple-braced `{{ }}` is used to escape literal braces in CSS/JS — keep this pattern. Lone `{` / `}` characters not paired with another are treated as f-string interpolation and will fail.
- **Project has no automated tests.** Verification is "regenerate HTML, open in a browser, check at the listed viewport widths."
- **Fast HTML regeneration** during UI tasks (no re-scrape needed) — use this one-liner; it bypasses the Selenium webdriver init:

  ```bash
  python -c "
  import json, RailScraper
  s = RailScraper.RailScraper.__new__(RailScraper.RailScraper)
  s.config = json.load(open('config.json'))
  data = json.load(open('timetable_data.json'))
  open('timetable.html','w').write(s.generate_html(data))
  print('regenerated timetable.html')
  "
  ```

  Use this after every UI/JS change; only Task 1 needs a full Selenium scrape.

- **Local browser verification:** open `timetable.html` directly via `file://` (the JS has no fetch dependency — `DATA` is inlined). For mobile width, use Chrome DevTools "Toggle device toolbar" → iPhone SE / 375px.
- **Generated artifacts** (`timetable.html`, `timetable_data.json`) are committed in this repo (CI commits them daily). Regenerate them locally and include in the relevant commit so the live site reflects changes immediately. CI will overwrite tomorrow regardless.

**Files:**


(`bottom: 2px` keeps the gradient above the 2px bottom border so the border stays crisp.)

- [ ] **Step 2: Add `updateTabBarFade()` helper in the JS block**

Inside the `<script>` block in `generate_html()`, add this function (place it near `renderTabs`, before `renderAll`):

```js
        function updateTabBarFade() {{
            const bar = document.getElementById('tabBar');
            const wrap = document.getElementById('tabBarWrap');
            if (!bar || !wrap) return;
            const atEnd = bar.scrollLeft + bar.clientWidth >= bar.scrollWidth - 1;
            const overflowing = bar.scrollWidth > bar.clientWidth + 1;
            wrap.classList.toggle('scrolled-end', atEnd || !overflowing);
        }}

        let _fadeRaf = 0;
        function scheduleFadeUpdate() {{
            if (_fadeRaf) return;
            _fadeRaf = requestAnimationFrame(() => {{
                _fadeRaf = 0;
                updateTabBarFade();
            }});
        }}
```

`overflowing` check hides the fade when content fits (e.g., desktop) since there's nothing to discover.

- [ ] **Step 3: Wire scroll/resize listeners in `init()`**

Find the `init()` function (around the bottom of the `<script>` block):

```js
        function init() {{
            loadPreferences();
            renderAll();
            setInterval(() => {{
                renderTimetable();
            }}, 60000);
        }}
```

Replace with:

```js
        function init() {{
            loadPreferences();
            renderAll();
            const bar = document.getElementById('tabBar');
            if (bar) bar.addEventListener('scroll', scheduleFadeUpdate, {{ passive: true }});
            window.addEventListener('resize', scheduleFadeUpdate);
            setInterval(() => {{
                renderTimetable();
            }}, 60000);
        }}
```

## Task 4: Reveal active tab via `scrollIntoView` on render

**Files:**
- Modify: `RailScraper.py` — `renderTabs()` JS function inside `generate_html()` (around RailScraper.py:601-616).

- [ ] **Step 1: Replace `renderTabs()`**

Find:

```js
        function renderTabs() {{
            const bar = document.getElementById('tabBar');
            bar.innerHTML = '';
            DATA.route_pairs.forEach((pair, i) => {{
                const tab = document.createElement('div');
                tab.className = 'tab' + (i === activePairIndex ? ' active' : '');
                tab.textContent = pair.label;
                tab.onclick = () => {{
                    activePairIndex = i;
                    activeDirectionIndex = pairDirections[i] ?? 0;
                    saveManualChoice();
                    renderAll();
                }};
                bar.appendChild(tab);
            }});
        }}
```

Replace with:

```js
        function renderTabs() {{
            const bar = document.getElementById('tabBar');
            bar.innerHTML = '';
            let activeEl = null;
            DATA.route_pairs.forEach((pair, i) => {{
                const tab = document.createElement('div');
                tab.className = 'tab' + (i === activePairIndex ? ' active' : '');
                tab.textContent = pair.label;
                tab.onclick = () => {{
                    activePairIndex = i;
                    activeDirectionIndex = pairDirections[i] ?? 0;
                    saveManualChoice();
                    renderAll();
                }};
                bar.appendChild(tab);
                if (i === activePairIndex) activeEl = tab;
            }});
            if (activeEl) {{
                activeEl.scrollIntoView({{ inline: 'center', block: 'nearest', behavior: 'auto' }});
            }}
            scheduleFadeUpdate();
        }}
```

`behavior: 'auto'` (no animation) — keeps initial paint snappy. `inline: 'center'` centers the active tab horizontally; near the start/end of the list it falls back to nearest edge automatically.
